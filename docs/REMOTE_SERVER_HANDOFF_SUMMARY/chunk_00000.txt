## TROUBLESHOOTING

### Issue: Cannot Access Server

**Check connectivity:**
```powershell
Test-Connection -ComputerName 10.0.0.157 -Count 4
```

**Verify VPN/network connection**
- Ensure connected to HPD network
- Check firewall settings
- Try pinging server

### Issue: File Copy Fails

**Permission errors:**
- Network shares: Use manual RDP copy method
- Admin shares: Requires local administrator rights
- Solution: Copy via RDP session (see MANUAL DEPLOYMENT PROCESS above)

### Issue: Import Fails in ArcGIS Pro

**Symptoms:**
- "Unable to read file" error
- Zero records imported
- Schema errors

**Solutions:**
1. Use local path (C:\...) not network path (\\...)
2. Verify file format (.xlsx not corrupted)
3. Check file size (~70-74 MB)
4. Try importing first sheet explicitly
5. Convert to CSV if Excel import continues to fail:
   ```python
   import pandas as pd
   df = pd.read_excel("CAD_Consolidated_2019_2026.xlsx")
   df.to_csv("CAD_Consolidated_2019_2026.csv", index=False)
   ```

### Issue: Record Count Mismatch

**Expected: 716,420 records**

If different:
1. Check date range filter during import
2. Verify complete file copied (size check)
3. Check for import errors in ArcGIS Pro messages
4. Validate using SQL query:
   ```sql
   SELECT COUNT(*) FROM CAD_Consolidated_2019_2026
   ```

### Issue: Broken Data Sources After Update

**Solution:**
1. Right-click broken layer → Properties → Source
2. Click "Set Data Source"
3. Browse to new consolidated file
4. Apply and save project

**Or use repair script:**
```
10_Projects/ESRI/2026_dashboards/04_Fix_Broken_References/02_Repair-BrokenReferences.ps1
```
