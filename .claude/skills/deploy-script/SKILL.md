# Deploy Script to RDP Server

Generate deployment commands for copying a script to HPD2022LAWSOFT and setting it up.

## Arguments

- `$0` -- Local script path (e.g., `scripts/monitor_dashboard_health.py`) (required)
- `$1` -- Optional: `--schedule` flag
- `$2` -- Optional: Time in `HH:MM` format for Task Scheduler (e.g., `01:30`)

## Step 1 -- Read and Validate the Script

Read the script at `$0` to understand:
- What it does (docstring, comments)
- Whether it requires `arcpy` (determines Python interpreter)
- What input files/paths it references
- What output it produces (logs, reports)
- Whether it has CLI arguments

## Step 2 -- Generate Deployment Commands

### Copy to server

Determine the target directory based on script purpose:
- ArcGIS/ESRI scripts -> `C:\HPD ESRI\04_Scripts\`
- Pipeline automation scripts -> `C:\ESRIExport\Scripts\`
- Utility/monitoring scripts -> `C:\HPD ESRI\04_Scripts\`

```powershell
# Option A: Copy via OneDrive sync (if repo is synced)
$source = "C:\Users\carucci_r\OneDrive - City of Hackensack\02_ETL_Scripts\cad_rms_data_quality\<script_path>"
$dest = "C:\HPD ESRI\04_Scripts\<script_name>"
Copy-Item -Path $source -Destination $dest -Force
Write-Host "Copied to $dest"
Get-Item $dest | Select-Object Name, Length, LastWriteTime

# Option B: If OneDrive not synced on server, paste content directly
# (Show the user the script content to paste into a file on the server)
```

### Verify the copy

```powershell
# Check the file exists and has content
$script = "<target_path>"
if (Test-Path $script) {
    $info = Get-Item $script
    Write-Host "OK: $($info.Name) - $($info.Length) bytes - $($info.LastWriteTime)"
} else {
    Write-Host "ERROR: File not found at $script"
}
```

- What output to look for

## Step 4 -- Create Scheduled Task (if --schedule provided)

Only generate this section if `$1` is `--schedule` and `$2` is provided.

```powershell
# Create directories
New-Item -ItemType Directory -Path "C:\ESRIExport\logs" -Force

# Define the task
$taskName = "<TaskName>"  # Derive from script name
$py = "C:\Program Files\ArcGIS\Pro\bin\Python\envs\arcgispro-py3\python.exe"
$script = "<target_path>"
$workDir = "<parent_directory_of_script>"

$action = New-ScheduledTaskAction `
    -Execute $py `
    -Argument "`"$script`"" `
    -WorkingDirectory $workDir

$trigger = New-ScheduledTaskTrigger -Daily -At "$2"

$settings = New-ScheduledTaskSettingsSet `
    -ExecutionTimeLimit (New-TimeSpan -Minutes 30) `
    -StartWhenAvailable

Register-ScheduledTask `
    -TaskName $taskName `
    -Action $action `
    -Trigger $trigger `
    -Settings $settings `
    -Description "Auto-generated by deploy-script skill"

# Set to run as administrator with highest privileges
$cred = Get-Credential -UserName "HPD\administrator" -Message "Task Scheduler password for $taskName"
schtasks /change /tn $taskName /ru "HPD\administrator" /rp $cred.GetNetworkCredential().Password /rl HIGHEST

# Verify
Get-ScheduledTask -TaskName $taskName |
    Select-Object TaskName,
        @{n='User';e={$_.Principal.UserId}},
        @{n='RunLevel';e={$_.Principal.RunLevel}},
        @{n='Executable';e={$_.Actions[0].Execute}},
        @{n='Arguments';e={$_.Actions[0].Arguments}} |
    Format-List
```

### Test the scheduled task

```powershell
Start-ScheduledTask -TaskName $taskName
Start-Sleep -Seconds 90
Get-ScheduledTaskInfo -TaskName $taskName | Select-Object LastRunTime, LastTaskResult
# LastTaskResult should be 0
```

## Important Notes

- **Always use `python.exe`**, not `pythonw.exe` (the latter hides errors and was the root cause of a prior pipeline failure)
- **Run as `HPD\administrator`** with RunLevel Highest -- SYSTEM account cannot access ArcGIS licenses
- **Working directory matters** for scripts that use relative paths or import from local packages
- **Do not modify `task.py`** files auto-generated by ArcGIS Pro -- they get overwritten on re-export
