KEY DESIGN DECISIONS
=============================================================================

1. CONFIGURATION-DRIVEN ARCHITECTURE
   - All paths in YAML config files (no hardcoded paths)
   - ${variable} expansion for flexible path management
   - Separate configs for schemas, validation, and sources

2. REFERENCE, DON'T DUPLICATE
   - Schemas loaded from 09_Reference/Standards (not copied into repo)
   - schema_loader.py provides centralized access
   - Reduces maintenance burden and ensures single source of truth

3. MODULAR ENGINE DESIGN
   - FieldNormalizer: Standalone normalization engine
   - ValidationEngine: Standalone validation engine
   - QualityScorer: Standalone scoring engine
   - Each can be used via Python import or CLI
   - consolidate_cad.py orchestrates all engines

4. 6-STEP PIPELINE ARCHITECTURE
   Step 1: Load configuration and source files
   Step 2: Normalize fields (FieldNormalizer)
   Step 3: Validate data (ValidationEngine)
   Step 4: Calculate quality scores (QualityScorer)
   Step 5: Deduplicate records
   Step 6: Export with metadata

5. QUALITY-FIRST DESIGN
   - FAIL if validation error_count > 0
   - FAIL if quality score < threshold (default: 95 for consolidation)
   - Quality tier embedded in output filename
   - Comprehensive metadata JSON output

6. PRODUCTION-READY FEATURES
   - Vectorized operations for performance
   - Comprehensive logging with timestamps
   - Audit trail (file hashes, metadata)
   - Gap detection (missing dates, low-count days)
   - Multiple export formats (CSV, JSON, XLSX)
   - CLI interfaces for all engines
   - Exit codes (0=success, 1=validation failure, 2=config error, 3=runtime error)
